SpringSecurity入门

发表于 2021-04-20 更新于 2021-04-21

SpringSecurity

Spring Security 是一个比 Shiro 优秀很多的权限管理框架，但是重量级、配置繁琐、门槛高这些问题一直困扰着 Spring Security 的开发者，也让很多初学者望而却步。直到 Spring Boot 横空出世，这些问题统统都得到缓解。

在 Spring Boot 或者 Spring Cloud 中，如果想选择一个权限管理框架，几乎毫无疑问的选择 Spring Security，Shiro 在这个环境下已经不具备优势了。

SpringSecurity介绍

Shiro优缺点：

  • 轻量
  • 简单
  • 易于集成
  • 但是对OAuth2支持不够，在SpringBoot中无法充分展示自己的优势。

Security优缺点：

  • 重量级
  • 配置繁琐
  • 门槛高

Shiro与SpringSecurity核心功能：

  • 认证
  • 授权

SpringSecurity实现核心

SpringSecurity实现的核心内容为一组过滤器链，项目启动后会自动配置。

最核心的过滤器：Basic Authentication Filter —> 实现认证用户身份。

比如对于username password认证过滤器来说，它的功能主要为：

  • 检查一个请求是否为登录请求
  • 是否包含username以及password
  • 如果不满足，放行给下一个过滤器
    • 下个过滤器会按照自身职责判定是否包含自身需要的信息，basic的特征就是在请求头中有 Authorization:Basic eHh4Onh4 的信息。中间可能还有更多的认证过滤器。最后一环是 FilterSecurityInterceptor，这里会判定该请求是否能进行访问rest服务，判断的依据是 BrowserSecurityConfig中的配置，如果被拒绝了就会抛出不同的异常（根据具体的原因）。Exception Translation Filter 会捕获抛出的错误，然后根据不同的认证方式进行信息的返回提示。
